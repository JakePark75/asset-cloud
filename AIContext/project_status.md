# 개인 자산관리 시스템 — 클라우드 전환 프로젝트

---

## 1. 프로젝트 개요

### 목적
로컬 PC에서 엑셀 + 파이썬으로 운영 중인 개인 자산 관리 시스템을 Oracle Cloud 기반으로 재구현한다.

### 1차 목표
기존 로컬 시스템의 핵심 기능을 클라우드에서 동일하게 동작하도록 재현한다.

### 이후 목표
다양한 기능 추가 및 성능 개선.

---

## 2. 현재 로컬 시스템 구조

### 파이썬이 하는 일
- KIS API (한국투자증권) / Yahoo Finance로 실시간 시세 수집
- win32com으로 열려있는 엑셀 시트에 직접 데이터 쓰기
- 텔레그램 봇으로 리포트 이미지 전송
- 시스템 트레이 UI로 백그라운드 실행 (pystray)
- 1분 단위 주기적 업데이트 (설정 가능)
- 과거 데이터 조회 모드 (특정 날짜 종가로 시트 업데이트)
- 매일 일간 누적 행 삽입 (insert_daily_row)

### 클라우드 전환 시 변경되는 것
- 엑셀 / win32com → DB + 웹 대시보드로 대체
- 시스템 트레이 UI → 제거 (서버에서 상시 실행)
- 텔레그램 봇 → 유지 (형태는 추후 결정)

---

## 3. 엑셀 시트 구조

### 메인 시트 3개 (1차 클라우드 전환 대상)

#### 시트 1: 종목실시간
- **역할**: 전체 시스템의 시세 원천 데이터
- **사용자 입력값**: 종목명 / 티커 / 구분 (KR, NAS, AMS, ARC, IDX, FX)
- **파이썬이 채우는 값**: 현재가 / 등락률 / 업데이트시간 / 데이터시간
- **특이사항**: 이 시트가 투자포트 시트의 원천이 됨

#### 시트 2: 투자포트
두 개의 표로 구성됨.

**계좌통합종목표**
- 종목실시간 시트에서 시세 참조
- 종목별 수량 합산 (여러 계좌 통합)
- 레버리지 배수 계산 (현재는 종목명 텍스트 파싱으로 x1/x2/x3 구분 — 클라우드에서는 별도 컬럼으로 관리 예정)
- Exposure / 현금비중 / x1·x2·x3 비중 계산의 출처
- 사용자 입력값 없음 (전부 수식)

**계좌별표**
- 계좌통합종목표에서 시세 참조하여 평가액 계산
- **사용자 입력값**: 계좌명 / 종목명 / 수량 / 현금(예수금, 수동 입력)
- 계좌비중 / 투자비중은 수식으로 계산
- 셀 병합 구조 (계좌명이 여러 종목 행에 걸쳐 병합)

#### 시트 3: 자산변동
두 개의 영역으로 구성됨.

**일일실적누적표**
- 매일 오전 (미국장 마감 후, 한국장 시작 전) 수동으로 전일 행 마감
- insert_daily_row()로 새 행 삽입
- **사용자 입력값**: 입출금액 (입금 +, 출금 -) / 사유
- 수식 계산값: 자산총액 / NDX100 / 등락률 / Exposure / 현금비중 / x1·x2·x3 비중 / 보정수익률 / TWR 자산 / MWR 보조열
- **이 시트가 IRR, 알파, TWR, MDD 등 모든 수익성 지표의 원천 데이터**
- 구조: 최신 날짜가 위, 과거가 아래 (행이 위로 쌓임)

**대시보드**
- 일일실적누적표를 참조해서 계산한 요약 지표 표시
- 항목:
  - 자산 현황: 현재 총자산 / 입출금 총액 / 초기자본
  - 수익성 IRR: 연평균 IRR / 월평균 IRR / TWR 수익률 / TWR 월평균
  - 수익성 알파: 알파 수익률 / 월평균 알파 / 최근 30일 알파 (벤치마크: 나스닥100)
  - 리스크: 샤프 지수 / 소르티노 지수 / My MDD / 지수 MDD
  - 은퇴 플래닝: 은퇴 시점 / 은퇴 예상 자산 / D-day
- **사용자 입력값**: 은퇴 목표일 (복수 설정 가능)

### 제외 시트 (1차 범위 외)
- 계좌별 매수/매도 이력, 평단 관리 시트 — 엑셀에서 계속 관리

## 5. 주요 지표 계산 로직

### IRR (연평균)
- XIRR 기반
- 오늘 총자산을 시작점으로, 과거 입출금 이력과 합쳐서 날짜순 정렬 후 계산
- 원천 데이터: 자산변동 시트의 날짜(I열) / 자산총액(J열) / 입출금(X열)
- 월평균 IRR은 연평균에서 월할 환산
- 파이썬 구현: `numpy_financial` 또는 `scipy`의 xirr 사용 예정

### 알파
- 내 포트폴리오 수익률 - 벤치마크(나스닥100) 수익률
- 기준 시점(고정행) 대비 현재 비율 차이
- 최근 30일 알파도 동일 방식, 기준 시점만 다름

### Exposure
- 보유 종목별 (평가액 × 레버리지 배수) 합산 / 총자산
- 레버리지 배수: x1=1, x2=2, x3=3
- 현재 엑셀에서는 종목명 텍스트 파싱으로 배수 판단

### TWR (시간가중수익률)
- 입출금 영향을 제거한 순수 운용 수익률

## 6. 클라우드 전환 범위 (1차)

### 구현 대상
- 시세 수집 스케줄러 (KIS API / Yahoo Finance)
- DB (종목마스터 / 포지션 / 시세 / 일일누적 / 입출금이력)
- 웹 대시보드 (대시보드 지표 표시 + 사용자 입력)
- 텔레그램 봇 (리포트 발송, 1차 포함 / 우선순위 최하위)
- insert_daily_row 스케줄러 자동화 (매일 오전 7~8시 중 고정 시각)
- 기존 일일자산누적 데이터 (2025-06-19~) DB 일괄 이전, 프로젝트 완료 시점에 최종 반영

### 제외 대상 (1차)
- 매수/매도 이력, 평단 관리
- 과거 데이터 일괄 조회 기능 (추후 추가 검토)
